## CDU003 Facturación Electrónica

Sus expandidos serían:

- CDU003.1 Generación del Borrador de Factura
- CDU003.2 Validación
- CDU003.3 Certificación (FEL)
- CDU003.4 Notificación Automática de Emisión
- CDU003.5 Sincronización Contable y de Cobro
- CDU003.6 Consultar Facturas
- CDU003.7 Visibilidad Cobros
- CDU003.8 Pago de Facturas
- CDU003.9 Pagos

| Nombre            | Generación del Borrador de Factura                                                                                                                                                                                                                                                                                                                                                                                 |
| Código            | CDU003.1                                                                                                                                                                                                                                                                                                                                                                                                           |
| Actores           | Piloto                                                                                                                                                                                                                                                                                                                                                                                                             |
| Descripción       | Se genera un borrador de la factura con datos capturados anteriormente.                                                                                                                                                                                                                                                                                                                                            |
| Precondiciones    | El piloto debe marcar la carga como “Entregada”                                                                                                                                                                                                                                                                                                                                                                    |

| Reglas de negocio | El piloto tiene un tiempo de espera razonable para la actualización del estado<br>El borrador debe de tener toda la información necesaria                                                                                                                                                                                                                                                                          |
| Reglas de calidad | EL borrador debe de ser claro y conciso con la información<br>Debe tener todos los campos completos                                                                                                                                                                                                                                                                                                                |

| Nombre            | Validación                                                                                                                                                                                                                                                                                                                                                                                                  |
| Código            | CDU003.2                                                                                                                                                                                                                                                                                                                                                                                                    |
| Actores           | Agente financiero                                                                                                                                                                                                                                                                                                                                                                                                        |
| Descripción       | El sistema envía los datos del borrador para validar diferentes campos detallados mas adelante                                                                                                                                                                                                                                                                                                              |
| Precondiciones    | Debe de existir un borrador de la factura.                                                                                                                                                                                                                                                                                                                                                                  |
| Post Condiciones  | Se validan campos que contiene el borrador según métricas de calidad                                                                                                                                                                                                                                                                                                                                        |
| Flujo principal   | 1 Se recibe el borrador<br>2 Se valida el borrador<br>2.1 Verificar que el NIT sea válido (13 caracteres) 2.2 Validar que no faltan campos obligatorios<br>2.2.1 Fecha<br>2.2.2 Moneda (GTQ)<br>2.2.3 Descripción del servicio<br>2.2.4 Desglose correcto del IVA                                                                                                                                           |
| Flujos alternos   | FA1: El NIT es invalido<br> FA1.1 Se notifica a el departamento que el NIT es invalido<br>FA1.2 Se solicita el cambio de NIT a uno valido<br>FA1.3 Se vuelve a solicitar la validación<br>FA1.2.4 Vuelve a el flujo principal.<br><br>FA2: Falta de campos<br>FA2.1 Se solicita rellenar campos faltantes en el borrador<br>FA2.2 Se vuelve a solicitar la validación<br> FA2.3 Vuelve a el flujo principal |
| Reglas de negocio | NIT sea válido (13 caracteres) y que el formato cumpla con las reglas de la SAT<br>Fecha valida<br>Moneda<br>Desglose correcto del IVA correspondiente a la legislación tributaria de Guatemala                                                                                                                                                                                                             |
| Reglas de calidad | Rápido para el análisis del borrador<br>Formato adecuado luego de la validación de los datos                                                                                                                                                                                                                                                                                                                |
